Refactor: Modular AIXCL structure

This refactoring transforms AIXCL from a monolithic script into a modular,
maintainable CLI structure with comprehensive testing and documentation.

## Major Changes

### Structure
- Extract monolithic script into modular CLI structure
- Create cli/, lib/, services/, completion/, docs/, tests/ directories
- Implement nested command structure (aixcl stack start)
- Move Docker Compose files to services/ directory

### Functionality
- Extract all CLI modules (stack, service, models, council, dashboard, utils)
- Extract shared libraries (common, docker_utils, color, logging, env_check, etc.)
- Implement full council module with configure, status, and list commands
- Preserve all existing functionality

### Code Cleanup
- Remove unused LLM council frontend code
- Remove temporary refactoring files
- Clean up duplicate and obsolete files
- Fix circular dependencies

### Testing
- Add comprehensive baseline test suite
- Add API endpoint test suite
- Add Continue plugin integration test
- Create quick test for fast validation

### Documentation
- Create CLI specification (docs/CLI_SPEC.md)
- Create usage guide (docs/USAGE.md)
- Create migration guide (docs/MIGRATION.md)
- Update test documentation

### Integration
- Verify Continue plugin integration
- Confirm database persistence works
- Test all API endpoints
- Validate OpenAI-compatible API

## Breaking Changes

- Command structure changed: `aixcl start` → `aixcl.sh stack start`
- Utils commands: `aixcl check-env` → `aixcl.sh utils check-env`
- All other commands remain similar (just use `aixcl.sh` instead of `aixcl`)

## Benefits

- Modular code structure (easier to maintain and extend)
- Comprehensive test suite (serves as working specification)
- Better error handling with color-coded output
- Complete documentation
- Removed unused code (frontend, temporary files)
