# AI Assistant Instructions

When working on AIXCL, follow these instructions to ensure consistent workflow and formatting.

## Development Workflow

**CRITICAL: Always follow the Issue-First Development workflow.**

1. **Create an issue first** - Never start coding without creating an issue first
   ```bash
   gh issue create --title "Brief description" --body "Detailed description"
   ```

2. **Create a branch** - Use descriptive branch names
   ```bash
   git checkout -b issue-<number>/<short-description>
   ```

3. **Make changes and commit** - Use conventional commit format
   ```bash
   git commit -m "type: Description

   Fixes #<number>"
   ```

4. **Push and create PR** - Always reference the issue
   ```bash
   git push -u origin <branch-name>
   gh pr create --title "Fix: Title (#<number>)" --body "Fixes #<number>"
   ```

## Formatting Rules

**IMPORTANT: Use plain text formatting to avoid encoding issues.**

### ✅ DO Use:
- Markdown checkboxes: `- [x]` for completed items
- Standard markdown: `**bold**`, `*italic*`, `` `code` ``
- Plain ASCII characters
- Numbered lists: `1.`, `2.`, `3.`

### ❌ DON'T Use:
- Unicode checkmarks: `✓`, `✔`, `✅` (these appear garbled as "âce...")
- Emoji in technical documentation
- Special Unicode characters that may not render consistently
